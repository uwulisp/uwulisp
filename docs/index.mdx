## Why pi-lisp?

<CardGrid>
  <Card title="Lisp at the surface" icon="seti:default">
    Write definitions, lambdas, conditionals, quotes, macros, lists, strings,
    integers, floats, booleans, and evaluator experiments using familiar parenthesized forms.
  </Card>
  <Card title="Cubical ideas inside" icon="puzzle">
    Work with intervals, path types, composition, dependent functions,
    dependent pairs, universes, variables, and Glue types.
  </Card>
  <Card title="Assembly when needed" icon="rocket">
    Use inline x86-64 NASM-style instruction lists or external assembly files
    and receive the value returned through RAX.
  </Card>
  <Card title="Two mascots, one language" icon="heart">
    Lispy the cat and Lispu the puppy give the project its softer edge:
    part guides, part guardians, fully committed to balanced parentheses.
  </Card>
</CardGrid>

<Aside type="note" title="File extensions">
  pi-lisp uses two file extensions depending on which language features you need:

  - **`.pi`** — standard pi-lisp source files (S-expressions, lambdas, macros, builtins, assembly).
  - **`.pic`** — pi-lisp_cubical source files, for code that uses cubical type theory primitives such as intervals, path types, composition, Π-types, Σ-types, and Glue types.

  File extension is autodetected — a `.pic` file is automatically run in cubical mode without needing a `--cubical` flag. Both extensions are recognised by their respective VS Code extensions above.
</Aside>

<Aside type="tip" title="Run the interpreter">
  Clone the interpreter repository and start it with Cargo:

  ```bash
  git clone https://github.com/pi-lisp/pi-lisp.git
  cd pi-lisp
  cargo run
  # if use nix
  nix shell github:pi-lisp/pi-lisp
  pi-lisp
  ```
</Aside>
